3.2.6 Manual Method

In this method we manually implement the Function-Args Pair Option manually instead of writing a novel syntax form for function definition with continuations.

(define (x-hash-ref args conts)

  (match-let(

              [(list table key) args]

              [(list continue-ok continue-key-not-found) conts]

              )

    (define ok #t)

    (define result (hash-ref table key (λ() (set! ok #f))))

    (cond

      [ok (apply (car continue-ok) (cdr continue-ok)]

      [else (apply (car continue-key-not-found) (cdr continue-key-not-found))]

      ))

The apply is verbose, so lets write a continuation function apply to shorten that up. The function name ’continue’ is already used so I will call it ’mc:apply’. mc:apply can also take a default argument to be used when the continuation is not a pair.

(define (x-hash-ref args conts)

  (match-let(

              [(list table key) args]

              [(list continue-ok continue-key-not-found) conts]

              )

    (define ok #t)

    (define result (hash-ref table key (λ() (set! ok #f))))

    (cond

      [ok (mc:apply continue-ok (Λ result))]

      [else (mc:apply continue-key-not-found)]

      )))

When we pass the continuation as a parameter we must first curry it against local variables, such as function parameters or specified output values. We will provide a function for this. Say (mc:curry a-continuation args) returns the curried function. We don’t need mc:curry in this example.

If we adopt a convention of making the first argument in the args list the name of the entire args list, and the same for conts, then we can bury the match-let work in a define-syntax function.

(mc:define (x-hash-ref (args table key) (conts continue-ok continue-key-not-found)

    (define ok #t)

    (define result (hash-ref table key (λ() (set! ok #f))))

    (cond

      [ok (mc:apply continue-ok (Λ result))]

      [else (mc:apply continue-key-not-found)]

      )))

We will add mc:define to the functions-lib module. However, we will change the approach to calling continuations, as discussed in the next section.
