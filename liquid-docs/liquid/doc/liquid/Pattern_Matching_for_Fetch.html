2.5.6 Pattern Matching for Fetch

We don’t pass SQL through this interface. Instead all operations have function names.

In order to fetch data we perform pattern matching. The user passes a pattern into the fetch routine, and then the fetch routine unifies the pattern against the specified table.

In the db-lib model, patterns apply to specific tables. See the dataplex model for multi-table actions.

[Implementation notes: At this time patterns apply to single tables. Also we really need to get code on the server side to do the unification, the current implementation is on the client side.]
