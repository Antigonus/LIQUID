2.2.7 To-do

Connection contexts have not yet been implemented. Currently the interface connects to postgresql using the user’s role and database via UNIX credentials.

Transaction contexts currently do not nest because the thread semaphores are done wrong. This should be fixed along with the connection context implementation.

Patterns are not fully implemented. Currently we only support the ’_’ that matches anything.

We really need to makes sure the to string conversion stuff makes for safe SQL. I kinda suspect that right now it doesn’t.

Deallocation for keysapces is not implemented. Also we need to add tests for the existance of stale values use (using keys that were deallocated).
